# Installation in an existing project

This guide is for those who **already have a running project** and want to add Mnemósine without breaking anything. If starting from scratch, use the template or see the [README](../README.md).

---

## Quick method (recommended)

```bash
# In your project root
curl -sSL https://raw.githubusercontent.com/jocsaacesar/mnemosine/main/en/install.sh | bash
```

The installer asks what to install (complete, essential, or pick components) and copies everything to the right places.

---

## Manual method

### 1. Download the framework

```bash
# Anywhere outside your project
git clone https://github.com/jocsaacesar/mnemosine.git
```

### 2. Copy the skills

```bash
# Inside your project folder
mkdir -p .claude/skills
cp -r /path/to/mnemosine/en/model/skills/*/ .claude/skills/
```

If you already have `.claude/skills/`, the framework skills are added alongside yours.

### 3. Copy the components you want

```bash
# Auditors (optional)
mkdir -p library/auditors
cp -r /path/to/mnemosine/en/model/library/auditors/*/ library/auditors/

# Standards (optional)
mkdir -p standards
cp /path/to/mnemosine/en/model/standards/*.md standards/

# Pipeline templates (optional)
mkdir -p library/project
cp -r /path/to/mnemosine/en/model/library/project/*/ library/project/
```

### 4. Create the folder structure

```bash
mkdir -p learning/{errors,context,fix,prevention}
mkdir -p plans/{backlog,operational,emergency,archive}
mkdir -p memory
mkdir -p infra/scripts
cp /path/to/mnemosine/en/model/infra/scripts/mnemosine-log.sh infra/scripts/
chmod +x infra/scripts/mnemosine-log.sh
```

### 5. Update your .gitignore

**Don't overwrite your `.gitignore`.** Add these lines at the end:

```gitignore
# Mnemósine — personal data
memory/
logs/
```

### 6. Run the onboarding

```bash
claude
# type: /get-started
```

The skill interviews you and generates a personalized `CLAUDE.md`.

---

## Caveats

### You already have a `CLAUDE.md`

`/get-started` will try to create a new one. Before running:

1. Rename temporarily — `mv CLAUDE.md CLAUDE.backup.md`
2. Run `/get-started` — let the skill generate the new one
3. Merge manually — take instructions from the backup and add to the new one

### You already have `.claude/skills/`

No problem. Skills are added alongside yours. Nothing is overwritten.

---

## What the framework creates in your project

After `/get-started`:

```
your-project/
├── .claude/skills/          <- Skills (you copied)
├── CLAUDE.md                <- Your AI's identity (generated by the skill)
├── memory/                  <- Created by /get-started (in .gitignore)
│   └── MEMORY.md
├── learning/                <- Incident records
├── plans/                   <- Work management
└── infra/scripts/           <- Telemetry script
```

---

## How to uninstall

```bash
# Remove skills
rm -rf .claude/skills/{start,wrap-up,get-started,create-skill,active-learning,approve-pr,telemetry,review-text,make-public,marketplace}

# Remove structure (optional — contains your data)
rm -rf memory/ learning/ plans/ library/ standards/ infra/

# Remove CLAUDE.md (your call — you may want to keep it)
# Remove .gitignore entries (edit manually)
```

No global residue. Your project goes back to how it was before.
