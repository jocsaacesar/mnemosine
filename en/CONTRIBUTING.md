# Contributing

Thank you for your interest in improving Mnemosine. This project exists to help creators build deeper and more intentional collaborations with AI — and contributions that serve this mission are welcome.

## Before contributing

1. **Read the [README](README.md)** to understand what this project is and isn't.
2. **Read the [guides](guias/)** to understand the core concepts (CLAUDE.md, memory, skills).
3. **Check [existing issues](../../issues)** to see if your idea or bug has already been discussed.

## What we accept

### Yes

- **New guides** — Practical guides on topics not yet covered (e.g.: multi-project setups, team collaboration patterns).
- **Template improvements** — Making starter templates clearer or more useful.
- **Fixes** — Correcting errors, typos or misleading information.
- **Examples** — Real examples of collaboration interfaces (sanitized — no personal data).
- **Translations** — Translating guides to other languages (open an issue first to coordinate).

### No

- **Prompt lists** — Lists of "useful prompts" without structure or explanation.
- **Unreviewed AI-generated content** — Content that appears to have been generated without human review.
- **Other tool configurations** — This project is about Claude Code. Configurations for other tools belong elsewhere.
- **Personal memory files** — Never include real memory files with personal information.

## How to contribute

### Reporting issues

Use the appropriate issue template:
- **Bug report** — Something in the guides or templates is wrong.
- **Feature request** — An idea for a new guide, template or improvement.
- **Question** — Need clarification on a concept or approach.

### Submitting changes

1. **Fork** the repository.
2. **Create a branch** with a descriptive name: `add-team-patterns-guide`, `fix-template-typo`.
3. **Make your changes** following the conventions below.
4. **Open a Pull Request** using the PR template. Explain *what* and *why*.

### Conventions

- **Language:** All files and documentation in **Portuguese (BR)**. English technical terms are accepted when there's no natural translation (e.g.: skill, Claude Code, CLAUDE.md).
- **Format:** Markdown, following existing structure and heading styles.
- **File names:** Lowercase, hyphens for spaces (`my-new-guide.md`, not `MyNewGuide.md`).
- **Commits:** Short, descriptive messages. One logical change per commit.
- **No AI-generated content:** If you use AI to help write, review and edit carefully. The standard is human-quality writing that teaches effectively.

## Code of Conduct

By participating in this project, you agree to follow the [Code of Conduct](CODE_OF_CONDUCT.md). Be respectful, constructive and focused on helping creators.

## Questions?

Open an issue with the **Question** template. There are no dumb questions — only unexplored ideas.
